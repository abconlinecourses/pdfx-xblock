PDFX Text Layer Fix - Testing Guide

⚠️ Important: This test should be run in the context of an Open edX course with a PDFX XBlock loaded.

Test Steps

Step 1: Open Browser Console

Press F12 or right-click and select "Inspect Element", then go to the Console tab.

Step 2: Run Comprehensive Diagnostic

Run the following command in the console:

diagnosePDFX()
Expected Result: Should auto-detect the block ID and show a comprehensive diagnostic report including:
  • PDF.js availability status
  • PDF instance detection
  • PDF document loading status
  • Text layer existence and content
  • Recommendations for any issues found

Step 3: Check Text Layer Status

If you know the specific block ID, you can run:

window.PDFX_TEXT_DEBUG.checkTextLayerStatus('pdfx_BLOCKID')
Expected Result: Should show detailed information about the text layer element, including dimensions, styles, and span count.

Step 4: Test Text Extraction

Test if text can be extracted from the PDF:

window.PDFX_TEXT_DEBUG.testTextExtraction('pdfx_BLOCKID')
Expected Result: Should show text extraction results including item count, styles, and sample text items.

Step 5: Force Text Layer Refresh

If the text layer is empty, try forcing a refresh:

window.PDFX_TEXT_DEBUG.forceTextLayerRefresh('pdfx_BLOCKID')
Expected Result: Should attempt to recreate the text layer with proper text content.

Step 6: Make Text Layer Visible (for debugging)

To see the text layer visually:

window.PDFX_TEXT_DEBUG.makeTextLayerVisible('pdfx_BLOCKID')
Expected Result: Text spans should become visible with red text and yellow background.

Step 7: Hide Text Layer (return to normal)

To return the text layer to normal (transparent):

window.PDFX_TEXT_DEBUG.hideTextLayer('pdfx_BLOCKID')
Expected Result: Text spans should return to transparent (normal state).

Common Issues and Solutions

Issue: "No PDF instance found"

Cause: The PDF initialization failed or is using a different system.

Solution: Check if the PDF is actually loaded and visible. The diagnostic will provide recommendations.

Issue: "No PDF document loaded"

Cause: PDF instance exists but the PDF file couldn't be loaded.

Solution: Check the PDF URL and network connectivity. Verify the PDF file is accessible.

Issue: "Text layer is empty"

Cause: Text extraction failed or text layer rendering failed.

Solution: Try the "Force Text Layer Refresh" command. Check if the PDF contains selectable text.

Debug Button Interface

The enhanced debug utilities also provide a visual debug interface that should appear automatically. Look for a dark debug panel in the top-right corner of the page with buttons for:

  • Full Diagnostic
  • Check Status
  • Test Extraction
  • Force Refresh
  • Make Visible
  • Hide Text
  • Check PDF.js

What the Fixes Address

1. PDF Instance Detection

Enhanced the debug utilities to find PDF instances created by different initialization systems (PdfxInit vs PdfxXBlock).

2. PDF Loading in PdfxInit

Fixed the broken loadPDF call in the PdfxInit function by implementing proper PDF loading and rendering.

3. Text Layer Creation

Enhanced text layer creation with multiple fallback methods and comprehensive error handling.

4. Comprehensive Diagnostics

Added detailed diagnostic tools to identify and troubleshoot text layer issues.
